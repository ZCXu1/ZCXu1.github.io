Solution

Analysis

这种题一般都往动态规划上去考虑，设一个dp数组，dp[i]表示以i结尾的所求字符串的长度，那么显然，如果s[i]=='('，则dp[i]=0，所以只需要考虑s[i]==')'，则s[i-1] == '(' or ')'，那么就分情况考虑，若s[i-1] == '('，即".......()"样式的字符串，则dp[i] = dp[i-2]+2，若s[i-1] == ')'，即"......))"样式的字符串，假设往前dp[i-1]个长度，再前面字符是x，也就是"....x(......))"，如果s[i - dp[i-1] - 1] == x == '('，也就是"...((.....))"，则这段长度len = dp[i-1] + 2，但是小心，闭合之后前面可能还有合法的字符串，"....(...)((......))"，所以实际上长度应该是dp[i-1] + 2 + dp[i-dp[i-1]-2]，如果x == ')'，那么显然根据dp[i-1]的定义，s[i] 是多余的，意思是x都找不到闭合的左括号，更别说s[i]了，所以dp[i] = 0。

拓扑排序

适用于有向图，有入度为0的点，无环。例如某个课程需要先修课，找出合理的修课顺序。

算法就是不断寻找入度为0的点。对于第一个入度为0的点，打印，然后把这个点和它的所有出去的边
都消除，也就是把它的next节点入度-1，重复这个过程。如果有多个入度为0的点，按批处理，每一批处理掉所有入度为0的点，
直到所有的节点都被打印了。所以这个算法需要hashmap记录每个点的入度，同时需要队列的顺序来保证
每一批的入度为0的节点被处理完，再处理下一批。

二叉树

前序遍历

算法
准备一个栈，首先把根节点压栈

  1. 从栈中弹出一个节点
  2. 打印该节点
  3. 先压右孩子再压左孩子（如果有的话）
  4. 循环以上三个操作，直到栈空

后续遍历

在前序遍历的基础上，需要第二个栈来收集一下根节点，并且调换一下压孩子的顺序

准备两个栈A和B，首先把根节点压栈A

  1. 从A栈中弹出一个节点
  2. 将该节点压入B栈
  3. 先压左孩子再压右孩子（如果有的话）
  4. 循环以上三个操作，直到A栈空
  5. 最后弹出B栈所有元素即为后序遍历

对于栈A，从栈A弹出然后进栈B的顺序为头右左，比如父节点D和左孩子E，右孩子F，D肯定是先进栈A，
弹出D后，压入E和F到A，所以出栈时先出F，再出E，所以进栈B的顺序是头右左，出栈B的时候就是
左右头，就是后序遍历了。

中序遍历

每棵子树，整棵树的左边界依次进栈，进完了之后依次弹出，打印，然后对弹出节点的右孩子重复这个过程。

按左边界可以对树划分，进栈的时候对每一颗子树左边界入栈，顺序是根->左，出栈的时候就是左->根，
然后对右树重复这个操作，其实整个操作没有右的概念，每个右都被划分为了左->根，但是右侧进行的晚，
所以整体是左->根->右，中序遍历。

二叉树的最近公共祖先

https://leetcode.cn/problems/er-cha-shu-de-zui-jin-gong-gong-zu-xian-lcof/

给定一个二叉树, 找到该树中两个指定节点的最近公共祖先。 百度百科中最近公共祖先的定义为：“对于有根树 T 的两个结点 p、q，最近公共祖先表示为一个结点 x，满足 x 是 p、q 的祖先且 x 的深度尽可能大（一个节点也可以是它自己的祖先）。”

直接看代码，注意basecase，遇到空返回空，遇到p返回p，遇到q返回q。

代码怎么来的？想清楚p和q的所有关系。第一种情况，p（或q）为p和q的公共祖先。
第二种情况，p和q不互为公共祖先，往上汇聚的时候才找到公共祖先。

对于这个算法的解释就是，如果root下的任意一子树，既没有p也没有q，那么一定会返回空。
画个图也很容易验证，对情况一，能得到正确答案。对于情况二，假设在root分叉了，左边一棵子树
含有p，右边的子树含有q，最后递归到basecase，左边子树肯定返回p，右边子树返回q，直接进入下面的if判断，最后返回root。

寻找二叉树的后继节点

二叉树的后继节点定义为：若在二叉树的中序遍历中存在A和B的关系为….AB….，则B是A的后继节点，
A是B的前驱节点。对于普通二叉树，直接中序遍历即可，那么如果给出一个特殊的二叉树定义，即在普通二叉树
基础上，对每一个节点，额外存储一个指向父节点的指针，是否有复杂度更低的算法。

那么算法就是，分两种情况，如果找X节点的后继节点

  1. 如果X有右孩子，那么X的右子树的最左节点就是X的后继，这就是中序遍历的顺序。
  2. 如果X没有右孩子，就往父节点寻找，假设父节点为Y，若X是Y的左孩子，那么X的后继节点
    就是Y，如果X是Y的右孩子那么就把Y赋值给X，重复上述过程，直到找到一个节点，让X是这个节点的左孩子。
    对于这个节点，X就是这个节点左子树的最右节点。
  3. 注意整棵树的最右边的节点，后继是null。

难的是要求时间复杂度O(N)，额外空间复杂度O(1)。
如果用额外空间就比较简单，就是哈希表，key是原节点，value是新分配的节点。

第一次遍历，遍历一遍链表，对每一个节点生成一个新节点，值一样，然后这两个节点分别存入map的k和v

第二次遍历，对每一个老节点，找出next和random指向谁，通过哈希表再新节点中可以知道指向哪两个新的节点。

map.get(node1).next = map.get(node1.next)

map.get(node1).random = map.get(node1.random)

如果不用额外的哈希表，算法是，每遍历一个老链表的节点，生成一个对应的新节点，并串在后面
比如对链表 1->2->3->null，先不管random指针，遍历完第一遍，生成1->1’->2->2’->3->3’->null。
接下来我们处理random指针假设最初1的random指向3，那么1’的random应该指向3’，
也就是3的next，这时候如果我们遍历到1，那么我们给1’的
random赋值为1.random.next，以此类推，这样这次遍历就把新链表的random指针处理完了。
最后一次遍历处理新链表的next指针即可。

核心思路就是两个节点配对相邻出现，方便操作。

快慢指针法，不用HashSet的算法要记住。

快指针F和慢指针S一开始都在链表头，F一次走两步，S一次走一步，最后一定在环内某个节点相遇，相遇时，F回到head处，S不动
然后F一次走一步，S一次走一步，再次相遇时就是入环节点。

证明：

  1. 设置fast和slow指针，fast每次走两步，slow每次走一步。

  2. fast和slow同时从头节点出发，若有环，若干步后必相遇与环内某处。因为fast总比slow多走一步，两者都在环中时，任何差距都会逐步追上。 slow入环后，两者在顺时针的步差上（假设链表在入环后沿着顺时针方向）至多为T-1（T为环内节点数），当slow入环时fast正好是slow的后继时步差为T-1。由于slow每走一步，fast和slow的步差减少1，因此slow至多走T-1步后，两指针相遇。

  3. 现在来做一些算式推导。

  4. 通过(式4)发现，对环外结点数a有 a % T = c，而slow指针此时再走c + 1个结点就会回到入环点。于是再设置一个位于起点处的指针（可以复用fast），令其与slow同步逐步向前，fast将在(n-1)*T + c步后到入环外结点的最后一个结点，也即入环点的前一个结点，与此同时slow也会绕环n-1次，并再走c步到达入环点在逆时针方向上的前一个结点，此时二者都再走一步即相遇与入环点。

两个链表是否相交

给定两个链表（有环或无环），若两个链表相交，返回相交的第一个节点，否则返回null。

先根据找环第一个节点的方法，找到两个链表第一个入环节点loop1和loop2
再根据loop1和loop2是否为空分四种情况讨论

若loo1和loop2都是null，则两个链表要么平行，要么相交为Y形。
head1开始遍历链表，到tail1，记录length1，同理记录tail2和length2，先判断tail1是否与tail2地址相等
如果不相等，则不相交，如果相等，则相交，开始找第一个节点。

如果length1==length2，则我们再从头遍历两个链表，相遇的节点就是所求的节点，因为后面的公共部分长度一样。
若length1>length2，让head1先走(length1-length2)步，转化为相等的情况即可。

这也是题目leetcode 160. 相交链表

若loop1和loop2有一个null，有一个不是null。这种情况，是不可能相交的。

所以只剩两个链表都有环， loop1和loop2都不为null。这种情况又分三种子情况。

第一种是，这两个链表独立。

第二种是，在环的入口相交，即loop1==loop2

第三种是loop1!=loop2


第二种好判断，loop1==loop2的时候就是第二种，然后把后面的环扔了，就变成了最开始Y形相交了。
区分第一种和第三种的方法是，让loop1继续走，在遇到自己之前如果遇到了loop2，就是第三种，如果没遇到loop2就是第一种。
