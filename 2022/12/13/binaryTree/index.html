二叉树

前序遍历

算法
准备一个栈，首先把根节点压栈

  1. 从栈中弹出一个节点
  2. 打印该节点
  3. 先压右孩子再压左孩子（如果有的话）
  4. 循环以上三个操作，直到栈空

后续遍历

在前序遍历的基础上，需要第二个栈来收集一下根节点，并且调换一下压孩子的顺序

准备两个栈A和B，首先把根节点压栈A

  1. 从A栈中弹出一个节点
  2. 将该节点压入B栈
  3. 先压左孩子再压右孩子（如果有的话）
  4. 循环以上三个操作，直到A栈空
  5. 最后弹出B栈所有元素即为后序遍历

对于栈A，从栈A弹出然后进栈B的顺序为头右左，比如父节点D和左孩子E，右孩子F，D肯定是先进栈A，
弹出D后，压入E和F到A，所以出栈时先出F，再出E，所以进栈B的顺序是头右左，出栈B的时候就是
左右头，就是后序遍历了。

中序遍历

每棵子树，整棵树的左边界依次进栈，进完了之后依次弹出，打印，然后对弹出节点的右孩子重复这个过程。

按左边界可以对树划分，进栈的时候对每一颗子树左边界入栈，顺序是根->左，出栈的时候就是左->根，
然后对右树重复这个操作，其实整个操作没有右的概念，每个右都被划分为了左->根，但是右侧进行的晚，
所以整体是左->根->右，中序遍历。

二叉树完全性检验

LeetCode 958

给定一个二叉树的root，确定它是否是一个完全二叉树。 在一个完全二叉树中，除了最后一个关卡外，所有关卡都是完全被填满的，并且最后一个关卡中的所有节点都是尽可能靠左的。它可以包含1到2h节点之间的最后一级 h 。

经典的判断完全二叉树的算法：

  1. 按层序遍历二叉树
  2. 当出现一个节点有右孩子但无左孩子时，直接判断为非完全二叉树
  3. 在满足条件2的情况下，若出现第一个左右孩子不全的节点（有左孩子无右孩子或无左右孩子），后续节点必须都是叶子节点，否则为非完全二叉树

平衡二叉树检验

LeetCode 110

给定一个二叉树，判断它是否是高度平衡的二叉树。本题中，一棵高度平衡二叉树定义为：一个二叉树每个节点 的左右两个子树的高度差的绝对值不超过 1 。

其实就是对于根为X的一棵树，它的左子树要平衡，它的右子树要平衡，除此之外，它的左右子树的高度差的绝对值要<=1。

那么对于X的左右子树，都需要两个信息：是否平衡和高度，所以对于左右子树的处理是一样的，可以递归。

递归的basecase显然就是空树，平衡的而且高度为0。

搜索二叉树检验

LeetCode 198

给你一个二叉树的根节点 root ，判断其是否是一个有效的二叉搜索树。有效二叉搜索树定义如下： 节点的左子树只包含 小于 当前节点的数。
节点的右子树只包含 大于 当前节点的数。
所有左子树和右子树自身必须也是二叉搜索树。

这个套路一样可以用，就是想从左树要什么信息，右树要什么信息，也就是树型DP套路。
其实就是对于根为X的一棵树，它的左子树要是搜索二叉树，
它的右子树是搜索二叉树，除此之外，它的左子树的最大值要比它小，右子树的最小值比它大。

那么对于左右子树要求不一样了，没法用递归，但事实上如果我们统一一下，对于每棵树，我们要
三个信息：是否是搜索二叉树，最大值和最小值即可。

但是要注意basecase，如果树本身就是空的，返回空即可，因为最大值和最小值不好设。
但这样的话写某一递归过程的时候就要判空了。
如果左边信息不空，就必须要左边最大值小于它，并且左边也是搜索二叉树，同理判断右边是否空。

满二叉树检验

套路也是一样的，和左树要高度和节点个数，和右树要高度和节点个数，在根节点汇总。

二叉树的最近公共祖先

https://leetcode.cn/problems/er-cha-shu-de-zui-jin-gong-gong-zu-xian-lcof/

给定一个二叉树, 找到该树中两个指定节点的最近公共祖先。 百度百科中最近公共祖先的定义为：“对于有根树 T 的两个结点 p、q，最近公共祖先表示为一个结点 x，满足 x 是 p、q 的祖先且 x 的深度尽可能大（一个节点也可以是它自己的祖先）。”

直接看代码，注意basecase，遇到空返回空，遇到p返回p，遇到q返回q。

代码怎么来的？想清楚p和q的所有关系。第一种情况，p（或q）为p和q的公共祖先。
第二种情况，p和q不互为公共祖先，往上汇聚的时候才找到公共祖先。

对于这个算法的解释就是，如果root下的任意一子树，既没有p也没有q，那么一定会返回空。
画个图也很容易验证，对情况一，能得到正确答案。对于情况二，假设在root分叉了，左边一棵子树
含有p，右边的子树含有q，最后递归到basecase，左边子树肯定返回p，右边子树返回q，直接进入下面的if判断，最后返回root。

寻找二叉树的后继节点

二叉树的后继节点定义为：若在二叉树的中序遍历中存在A和B的关系为….AB….，则B是A的后继节点，
A是B的前驱节点。对于普通二叉树，直接中序遍历即可，那么如果给出一个特殊的二叉树定义，即在普通二叉树
基础上，对每一个节点，额外存储一个指向父节点的指针，是否有复杂度更低的算法。

那么算法就是，分两种情况，如果找X节点的后继节点

  1. 如果X有右孩子，那么X的右子树的最左节点就是X的后继，这就是中序遍历的顺序。
  2. 如果X没有右孩子，就往父节点寻找，假设父节点为Y，若X是Y的左孩子，那么X的后继节点
    就是Y，如果X是Y的右孩子那么就把Y赋值给X，重复上述过程，直到找到一个节点，让X是这个节点的左孩子。
    对于这个节点，X就是这个节点左子树的最右节点。
  3. 注意整棵树的最右边的节点，后继是null。

序列化二叉树

剑指 Offer 37

请实现两个函数，分别用来序列化和反序列化二叉树。
你需要设计一个算法来实现二叉树的序列化与反序列化。这里不限定你的序列 / 反序列化算法执行逻辑，你只需要保证一个二叉树可以被序列化为一个字符串并且将这个字符串反序列化为原始的树结构。

这里我们使用前序遍历的方式来序列化二叉树，注意存储字符串时用”_”分割（包括null），用”#”来代替null。
前序遍历序列化就比较简单了，反序列化因为存储了空节点，建树的时候，先去建左子树，建完
左子树再去建右子树，对每一个节点都是这样，也是个递归，用队列存储序列化的字符串来建树更方便。

对折纸条（微软）

注意这题不光要总结出是中序遍历一个左边都是凹，右边都是凸的满二叉树，而且要注意复杂度，N限制了树的层，而且是满二叉树，然后用一个boolean变量限制打印哪个字符，既然左孩子节点都是凹，右孩子节点都是凸，代码里的先true后false，就是表示左右孩子的节点类型的。
